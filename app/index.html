LaunchFolio

One config file. Portfolio, resume, and README — always in sync.

Step 1 — Upload your resume optional

We'll extract your existing experience, skills, education, and certifications. Don't worry if it's out of date — you'll enrich it in the next step.

Step 2 — Add anything that's not in the resume optional

Paste anything here — recent projects, a new role that started last month, skills you've picked up, accomplishments worth adding, even a rough bullet list. This enriches the resume data rather than replacing it. We'll merge it intelligently.

Enrichment text takes precedence over the resume for fields that overlap (e.g. current role, recent skills). Older structured data from the PDF is kept where no enrichment is provided.

Step 3 — Review & correct

Pre-filled from your resume where possible. Override anything here — these always take precedence.

Step 4 — Generate

Produces a data.js and a ready-to-deploy portfolio ZIP. After deploying, just edit data.js and push — CI/CD keeps everything in sync.

Your portfolio is ready

Download the ZIP, push to GitHub, enable Pages. Live in 60 seconds.

Download ZIP
Deploy steps
  1. Unzip the downloaded file
  2. Create a GitHub repo named <yourusername>.github.io
  3. Push: git init && git remote add origin <repo-url> && git add . && git commit -m "Initial portfolio" && git push -u origin main
  4. GitHub repo → Settings → Pages → Source: main → Save
  5. Live at https://<yourusername>.github.io in ~60 seconds

After that, just edit data.js and push — CI/CD handles the rest.
